## 오늘 배운 것
```
  1. relative 과 absolute
     - relative (상대적): "내가 원래 있어야 할 자리"를 기준으로 움직여요.
      이동해도 원래 자리는 비워두지 않고 그대로 차지하고 있어서 다른 요소들이 밀리지 않아요.
     - absolute (절대적): "가장 가까운 부모(static이 아닌)"를 기준으로 움직여요.
      만약 부모 중에 기준이 없으면 화면 전체(Viewport)를 기준으로 삼고, 공중에 붕 뜨기 때문에 원래 자리를 차지하지 않아요.

  2. GNB(Global Navigation Bar) 연습.  
    1) 구조 설계 (HTML)  
     - 3단계 계층: main-menu (1단계) → sub-menu (2단계) → last-menu (3단계)로 이어지는 구조.

    2) 스타일링 전략 (CSS)  
     - 포지셔닝: position: relative (부모)와 position: absolute (자식)를 조합하여    
       하위 메뉴가 상위 메뉴 옆에 자연스럽게 붙도록 설계.

     - 상태 제어: display: none으로 숨겨두었다가, :hover 가상 클래스를 이용해 마우스를 올리면
       display: block으로 나타나게 하는 전형적인 드롭다운 구현 방식이에요.

    3) 코드의 특징  
     - Flexbox 사용: main-menu를 flex-direction: column으로 배치해서 세로형 메뉴 구조를 만드셨네요.
       (만약 가로형 메뉴를 원하신다면 이 부분을 row로 바꾸거나 수정이 필요할 수 있어요!)

     - hover를 활용해 마우스를 올리면 display: none을 display: block으로 바꾸어 메뉴가 나타나게 설계.

    4) display none
        구조: ul과 li를 중첩하여 3단계 드롭다운 메뉴를 구성하셨어요. (main → sub → last)

        배치: position: absolute를 사용해 하위 메뉴들이 부모 메뉴의 오른쪽(left: 150px)에 찰떡같이 붙게 만드셨어요.

        동작: :hover를 활용해 마우스를 올리면 display: none을 display: block으로 바꾸어 메뉴가 나타나게 설계하셨죠.

```
