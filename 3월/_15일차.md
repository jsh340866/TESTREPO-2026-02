1. 변화 감지하기 (Event Listener)
catRestorant.addEventListener('change', (e) => { ... })
풀이: 사용자가 카테고리 선택창(드롭다운)에서 다른 항목을 클릭하는 순간, 자바스크립트가 "앗! 값이 바뀌었네?" 하고 알아차리는 부분이에요.

e: 이벤트 정보를 담고 있는 바구니예요.

2. 선택한 값 가져오기
console.log(e.target.value);
const target = e.target.value;

3. 데이터 걸러내기 (Filter)
const selectItems = 중구맛집_data.filter((item) => {

target == 'all': 전체를 골랐다면 모든 데이터를 통과!
target.includes(item.FD_CS): 내가 고른 카테고리 이름이 데이터의 카테고리(FD_CS)와 일치하는지 확인해요.

*** 08_PROTOTYPE(참고영역) *** - 09_PROTOTYPE 참조

1. 생성자 함수 정의 (Person)
객체를 만들기 위한 설계도를 선언하는 단계예요.
function Person(name) { this.name = name; }
    new 키워드로 새로운 사람을 만들 때, 각 사람마다 고유하게 가질 이름(name)을 저장하도록 설정해요.

2. 기존 기능 재정의 (toString)
자바스크립트의 모든 객체가 기본으로 가진 조상님의 기능을 내 입맛에 맞게 개조하는 단계예요.
Person.prototype.toString = function() { ... }
    이제 Person으로 만든 객체에서 toString()을 호출하면 기본 동작 대신 승원님이 정의한 문구가 출력돼요.

3. 새로운 기능 확장 (helloWorld)
설계도에는 없었지만, Person 가족들이 공통으로 사용할 새로운 기술을 추가하는 단계예요.
Person.prototype.helloWorld = function() { ... }
    prototype에 추가했기 때문에 메모리를 아끼면서 모든 자식 객체들이 이 기능을 공유할 수 있어요.

4. 실제 객체 생성 (person1)
설계도를 바탕으로 **실제 데이터(인스턴스)**를 찍어내는 단계예요.
const person1 = new Person("홍길동");
    메모리 어딘가에 name: "홍길동"을 가진 실제 객체가 만들어져요.

5. 데이터 확인 (console.log)
만들어진 객체가 어떤 모습을 하고 있는지 검사하는 단계예요.
console.log(person1);
    객체 안에 name이 잘 들어있는지, 그리고 그 뒤에 숨겨진 조상님(__proto__)은 누구인지 콘솔창에서 확인할 수 있어요.

6. 재정의된 메서드 실행
개조했던 기능을 실제로 사용해보는 단계예요.
person1.toString();
    자바스크립트는 먼저 person1 안을 찾아보고, 없으면 조상님(prototype)에게 가서 만든 toString을 찾아 실행해요.

7. 확장된 메서드 실행
새로 추가했던 기술이 잘 작동하는지 최종 확인하는 단계예요.
person1.helloWorld();
    콘솔에 "홍길동 HELLO WORLD"가 찍히면서 모든 로직이 성공적으로 마무리됩니다!
