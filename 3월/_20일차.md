      *** 08_동기비동기(SYNC&ASYNC) ***
    - 동기 함수와 비동기 함수
    
    1. 메인스레드에서의 반복문  --> 첫번째로 출력되는 for문.
        for (let i = 0; i < 10; i++) {
          console.log("main thread...for i :", i)
        }
        console.log("첫번째 for문 종료..");
        console.log("---------------------------------")
    2. 비동기 함수(스레드 분할 처리 - 실제로 JS에서는 싱글 쓰레드만 지원/ 구조적으로 스레드 분할불가)
                              --> 두번째로 만든 for문 이지만 setTimeout() 비동기 함수로 마지막에 처리!
        setTimeout(() => {
          for (let i = 0; i < 10; i++) {
            console.log("Async thread...for i :", i)
          }
          console.log("두번째 for문 종료..");
        }, 5);
    3. 메인스레드에서의 반복문 --> 세번째로 만든 for문, 출력은 두번째로 출력!
        for (let i = 0; i < 10; i++) {
          console.log("main thread...for i :", i)
        }
        console.log("세번째 for문 종료..");
        console.log("---------------------------------")


          *** 04_비동기_Promise ***
    new Promise((resolve, reject) => {}).then((response)=>{}) // 비동기 작업 성공시(resolve)
                                        .catch((error)=>{})   // 비동기 작업 실패시(reject)
                                        
    1. 비동기 기본 함수
    function sub_process_executor(val) {
      return new Promise((resolve, reject) => {
        setTimeout(() => {
          resolve(val);
        }, 2000)
      })
    }

        *** Async / Await 를 이용한 비동기 처리로직 순서 배치 ***

    - async 와 await 를 사용해 비동기 처리에 순서를 부여할 수있다.
    - 22번을 먼저 출력하고 2000ms(2초)후 11번을 출력
    async function sub_process_order() {

      await sub_process_executor(22).then((response) => {
        console.log('sub2', response);
        sub2ThreadEl.innerHTML += `sub2...${response}`;
      }).catch((error) => { })

      await sub_process_executor(11).then((response) => {
        console.log('sub1', response);
        sub1ThreadEl.innerHTML += `sub1...${response}`;
      }).catch((error) => { })
    }
    sub_process_order();
