    *** 02_Hosting ***
    호스팅이란 ..
       변수와 함수의 선언문을 코드의 맨 위로 끌어올리는 것처럼 동작하는 현상
    
    1. 변수 호스팅 (var vs let/const)
        var (너그러운 친구): 선언문이 위로 끌어올려지면서 undefined로 초기화까지 돼요.
                            그래서 선언 전에 불러도 에러는 안 나지만, 값은 몰라요.
        let, const (엄격한 친구): 선언문이 위로 올라가긴 하지만, 초기화가 안 돼요.
                                 그래서 선언 전에 쓰려고 하면 "아직 준비 안 됐어!"라며 에러를 내요.
    2. 함수 호스팅 (function vs Arrow Function)
        function h1() { ... } (전통적 방식): 함수 전체가 통째로 위로 올라가요. 그래서 코드 어디에서든 호출할 수 있는 마법이 일어나죠! 
        const h2 = () => { ... } (화살표 함수): 이건 '함수'라기보다 **'변수에 함수를 담은 것'**으로 취급해요.
                                               const 변수 호스팅 규칙을 따르기 때문에, 선언하기 전에는 절대 부를 수 없어요.

    *** 03_Scope ***
    1. 전역 스코프 (Global Scope)
    범위: 코드 전체, 어디서든 접근 가능해요.
    특징: 파일 맨 바깥에 선언하거나 var로 함수 밖에서 선언하면 전역 변수가 돼요.
         너무 많이 쓰면 이름이 겹쳐서 위험할 수 있어요!
    
    2. 함수 스코프 (Function Scope)
    범위: function 키워드로 만든 함수 내부에서만 유효해요.
    특징: var는 이 함수 스코프를 따라가요. 함수 밖에서는 안을 들여다볼 수 없지만, 함수 안에서는 밖(전역)을 볼 수 있답니다.
    
    3. 블록 스코프 (Block Scope)
    범위: 중괄호 { }로 감싸진 모든 곳 (if, for, while 등).
    특징: let과 const가 이 규칙을 철저히 지켜요. { } 밖으로 나가는 순간 변수는 사라져서 아주 안전해요!
    
    4. 렉시컬 스코프 (Lexical Scope)
    핵심: "태어난 곳이 고향이다!"
    의미: 함수를 어디서 실행했는지가 아니라, 어디에 정의(선언)했는지에 따라 상위 스코프가 결정돼요.
         코드를 적는 순간 이미 스코프가 정해진다고 해서 '정적 스코프'라고도 불러요.

    *** 04_Closure ***
    클로저(Closure)
    정의: 외부 함수가 끝나도, 내부 함수가 외부 함수의 변수를 끝까지 기억하고 있는 상태예요.
    용도 (정보 은닉): 변수를 아무나 못 건드리게 함수 안에 숨겨두고, 특정 함수로만 값을 수정하고 싶을 때 써요.
    특징 (데이터 보존): 함수를 호출할 때마다 값이 초기화되지 않고, 이전 상태를 그대로 유지할 수 있어요.


    *** 05_CallBack(중요) ***
    1. 콜백(Callback)의 진짜 의미
    쉽게 말해서 **"내가 실행하는 게 아니라, 남한테 '이따가 때 되면 실행해줘'라고 맡기는 함수"**예요.
    일반 함수: "지금 당장 실행해!" (Call)
    콜백 함수: "이 조건이 맞을 때, 혹은 이 작업이 끝났을 때 나중에 다시 불러줘!" (Call Back)

    2. 코드 속의 콜백 찾기
    1. 여기서 (e) => { ... } 이 부분이 바로 콜백 함수예요!
    boardEl.addEventListener('drop', (e) => { 
      e.preventDefault();
      console.log("드롭됐다!"); 
    });
    상황: 브라우저에게 "지금 당장 로그 찍어!"라고 하지 않았어요.
    약속: "사용자가 마우스를 'drop' 하는 순간(조건), 내가 준 이 함수를 **대신 실행(Callback)**해줘!"라고 예약만 걸어둔 거죠.
    실행: 실제로 마우스를 뗄 때, 브라우저가 승원님 대신 그 함수를 "똑똑" 두드려 실행시켜 줍니다.
    
    3. 왜 콜백을 쓰나요? (핵심 이유)
    유연성 (부품 갈아 끼우기):
    - 아까 만든 customMap(array, logic) 기억나시죠? customMap은 '반복문'이라는 틀만 가지고 있고,
      **실제 계산(콜백)**은 밖에서 갈아 끼울 수 있어요. (더하기를 줄지, 빼기를 줄지 마음대로!)
    비동기 (기다림):
    - 서버에서 데이터를 가져오거나, 사용자가 클릭할 때까지 프로그램이 멈춰있으면 안 되잖아요?
      그럴 때 **"다 되면 이거 실행해~"**라고 콜백을 던져놓고 다른 일을 할 수 있어요.
